1 전략 & 범위

본 문서는 두 가지 우선 기능 영역 — (A) 상품 카탈로그 & 검색 UX와 (B) 다국어/다통화 & 글로벌 배송 — 을 iHerb에서 벤치마킹하여 OPLE.COM에 적용하고, 이를 Shopify 커스텀 테마 및 앱 개발 방식으로 구현하는 상세 플로우를 담고 있습니다.

🎯 핵심 목표

iHerb 수준의 상품 발견성(findability)과 글로벌 쇼핑 경험을 Shopify 기반 OPLE.COM 스토어프론트에 이식

4 구현 플로우 A: 상품 카탈로그 & 검색 UX

4-1. 데이터 모델 설계 (Metaobjects)

iHerb식 풍부한 상품 정보를 Shopify의 Metaobjects로 정규화합니다.

# Metaobject: ingredient (성분)
- name_ko: string
- name_en: string
- dosage_unit: single-line
- allergen_flag: boolean

# Metaobject: product_extension (상품 확장)
- product: product reference
- ingredients: list<ingredient>
- suggested_use_ko: rich-text
- certifications: list<single-line>  (예: USDA Organic, Non-GMO)
- kfda_notice: rich-text               (식약처 고시사항)

4-2. 검색 UX 구현 단계

  1. Predictive Search 확장 — Shopify Search & Discovery 앱에서 동의어/유사어 사전 등록 (예: "오메가3" ↔ "EPA", "DHA", "피쉬오일")
  2. 커스텀 자동완성 UI — Liquid 테마에 Alpine.js 기반 드롭다운, 썸네일+가격+평점 미리보기 표시
  3. 검색 결과 페이지 — /search 템플릿 재작성, 좌측 패싯 필터(브랜드/성분/가격/평점) + 우측 상품 그리드
  4. No-result 폴백 — 커스텀 앱이 오타 교정 + 연관 카테고리 추천
  5. 검색 로그 수집 — 앱에서 쿼리/클릭 저장 → 향후 동의어 사전 자동 확장

4-3. 필터 & 카테고리 구현

  1. Product Metafields 정의 — brand, form(정제/캡슐/분말), serving_size, vegan, allergens를 metafield로 정의하여 Storefront Filter에 노출
  2. Collection 3-depth 구조 — 대카테고리(비타민) → 중카테고리(비타민D) → 소카테고리(비타민D3 5000IU) 를 Smart Collection 규칙으로 자동 생성
  3. 동적 Collection 템플릿 — Liquid의 {% paginate %} + collection.filters 활용, AJAX로 필터 토글 시 URL 쿼리 업데이트 & 부분 리렌더
  4. 상품 비교 기능 — 커스텀 앱이 localStorage에 variant ID 저장, /pages/compare 커스텀 페이지에서 metafield 기반 비교 표 렌더링

5 구현 플로우 B: 다국어/다통화 & 글로벌 배송

5-1. Shopify Markets 설정

  1. Market 정의 — Primary(KR), International(US/JP/CN) 4개 Market 생성, 각 Market에 통화·언어·도메인 서브패스 매핑 (ople.com/kr, /en, /jp, /cn)
  2. 통화 설정 — Local currencies 활성화, 가격 반올림 룰 정의 (JPY는 소수점 없음, KRW는 100원 단위)
  3. Geolocation 앱 설치 — 방문자 IP 기반 자동 리다이렉트 + 상단 국가/언어 셀렉터 배너

5-2. 다국어 콘텐츠 플로우

  1. Translate & Adapt 앱으로 테마 텍스트·상품명·설명·metafield 자동 번역 초안 생성
  2. 커스텀 앱에 번역 동기화 잡 — 관리자가 커스텀 앱 UI에서 번역본 리뷰/수정 → Admin API로 Shopify에 저장
  3. URL 구조 — hreflang 자동 삽입, Liquid에서 {{ request.locale.iso_code }} 기반 canonical 분기

5-3. 글로벌 배송 & 관세 구현

  1. Shipping Zones 재편 — 국내(CJ대한통운), 동아시아(JP/CN 해외특송), 북미(DHL/UPS) 로 존 분리 및 weight-based rate 설정
  2. Carrier Service API — 커스텀 앱이 외부 운송사 API와 통신하여 실시간 배송료 반환, 체크아웃 화면에 주입
  3. HS코드 & 관세 매핑 — 상품별 HS코드 metafield 등록, 앱이 Shopify Markets Duties API로 사전 관세 계산 → PDP 및 장바구니에 DDP 금액 표시
  4. 결제수단 라우팅 — Shopify Payments(해외 카드) + PortOne(국내 카카오페이/네이버페이/무통장) + PayPal(해외 대체)
