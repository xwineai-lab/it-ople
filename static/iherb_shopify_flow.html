4-2. iHerb 매핑 & 추가 데이터 소스 확보

현재 IHerbMapping의 4단계 매칭 방식을 개선하고, 외부 데이터 소스를 활용해 매핑 커버리지와 정확도를 끌어올립니다.

매칭 정확도 개선 전략

  1. 1차 – UPC/바코드 정확매칭 → IHerbProduct.upc_barcode 기반 결정론적 매칭 (score 1.0). OPLE 상품에 UPC 수집 필드 추가 필요.
  2. 2차 – 브랜드 + 제품명 + 용량 퍼지매칭 → RapidFuzz Token-Set Ratio + 용량/수량 정규식 추출 ("500mg, 120 Capsules")로 false-positive 감소.
  3. 3차 – 임베딩 유사도 (AI) → Sentence-Transformers (multilingual-e5-large)로 한/영 상품명 임베딩 후 코사인 유사도 Top-K 후보 제시.
  4. 4차 – 사람 검수 (manual) → 관리자 UI에서 Top-5 후보 비교 → verified 플래그 설정. 결정 로그를 학습 데이터로 축적.

설계 원칙

🌐 i18n First

언어·국가-의존 필드는 모두 별도 테이블로 분리. 신규 언어 추가 시 product_i18n 행만 추가하면 됨.

💱 국가별 가격/재고

product_price가 country_code+currency 단위로 저장. 관부가세·재고까지 국가별 관리.

🔗 Source 역추적

product_mapping이 iHerb/Amazon/제조사 등 다중 소스를 1:N으로 연결, 데이터 출처 감사(audit) 가능.

🧪 성분 정규화

성분 마스터(ingredient)를 별도 분리 → 알레르기·비건 필터, 성분별 상품 크로스셀 가능.

마이그레이션 전략 (현 DB → 신 DB)

  1. 신 스키마 생성 — 별도 PostgreSQL 스키마 catalog_v2에 Alembic 마이그레이션으로 테이블 생성
  2. brand/category/ingredient 마스터 추출 — 기존 Product.brand, IHerbProduct.category_path, ingredients_list를 중복제거(dedupe)하여 마스터 적재
  3. product_master 생성 — OPLE Product를 기준으로 UUID 발급, IHerbMapping이 verified=true인 건은 IHerbProduct 데이터로 보강
  4. product_i18n (ko/en) 분리 이관 — name_ko→ko row, name_en/description→en row; iHerb의 일/중 데이터가 있으면 추가 locale row 생성
  5. product_price 초기화 — KRW→country=KR, USD→country=US 기본값으로 시드. 이후 JP/CN 가격은 환율+마진 룰로 자동 산출
  6. 검증 쿼리 — 상품 수/평균 가격/카테고리 분포를 v1 vs v2 비교 리포트로 체크, 차이 < 0.1% 목표

4-4. Shopify 일괄 업로드 구조화

최적화된 신 DB를 Shopify가 이해하는 형식으로 변환하여 대량 업로드합니다. Shopify는 CSV 임포트와 GraphQL Admin API의 Bulk Operations 두 경로를 제공합니다.

업로드 단계별 세부 플로우

  1. ETL 쿼리 실행 — product_master + i18n(ko) + price(KR) + media + mapping 조인, verified=true 상품만 추출 (1차 배치)
  2. Shopify JSONL 변환 — 각 row를 Shopify의 ProductInput 스키마로 매핑, metafields 배열 구성, media 배열 구성
  3. Staged Upload — stagedUploadsCreate로 GCS URL 받고 JSONL 업로드 (상품 1,000건 ≒ 1~2MB)
  4. Bulk Mutation 실행 — bulkOperationRunMutation으로 productCreate 일괄 호출, 60초 단위 상태 폴링
  5. 번역 등록 — 생성된 Product GID 회수 후 locale별 translationsRegister로 en/ja/zh-CN 번역 주입
  6. Market별 가격 업데이트 — productVariantsBulkUpdate로 US/JP/CN 시장 가격(contextualPricing) 세팅
  7. 검증 & 재시도 — 실패 rows를 dead letter 테이블에 적재, 배치 단위 재실행. 샘플 10개 수동 QA.

5 구현 플로우 A: 상품 카탈로그 & 검색 UX

5-1. 데이터 모델 설계 (Metaobjects)

iHerb식 풍부한 상품 정보를 Shopify의 Metaobjects로 정규화합니다.

# Metaobject: ingredient (성분)
- name_ko: string
- name_en: string
- dosage_unit: single-line
- allergen_flag: boolean

# Metaobject: product_extension (상품 확장)
- product: product reference
- ingredients: list<ingredient>
- suggested_use_ko: rich-text
- certifications: list<single-line>  (예: USDA Organic, Non-GMO)
- kfda_notice: rich-text               (식약처 고시사항)

5-2. 검색 UX 구현 단계

  1. Predictive Search 확장 — Shopify Search & Discovery 앱에서 동의어/유사어 사전 등록 (예: "오메가3" ↔ "EPA", "DHA", "피쉬오일")
  2. 커스텀 자동완성 UI — Liquid 테마에 Alpine.js 기반 드롭다운, 썸네일+가격+평점 미리보기 표시
  3. 검색 결과 페이지 — /search 템플릿 재작성, 좌측 패싯 필터(브랜드/성분/가격/평점) + 우측 상품 그리드
  4. No-result 폴백 — 커스텀 앱이 오타 교정 + 연관 카테고리 추천
  5. 검색 로그 수집 — 앱에서 쿼리/클릭 저장 → 향후 동의어 사전 자동 확장

5-3. 필터 & 카테고리 구현

  1. Product Metafields 정의 — brand, form(정제/캡슐/분말), serving_size, vegan, allergens를 metafield로 정의하여 Storefront Filter에 노출
  2. Collection 3-depth 구조 — 대카테고리(비타민) → 중카테고리(비타민D) → 소카테고리(비타민D3 5000IU) 를 Smart Collection 규칙으로 자동 생성
  3. 동적 Collection 템플릿 — Liquid의 {% paginate %} + collection.filters 활용, AJAX로 필터 토글 시 URL 쿼리 업데이트 & 부분 리렌더
  4. 상품 비교 기능 — 커스텀 앱이 localStorage에 variant ID 저장, /pages/compare 커스텀 페이지에서 metafield 기반 비교 표 렌더링

6 구현 플로우 B: 다국어/다통화 & 글로벌 배송

6-1. Shopify Markets 설정

  1. Market 정의 — Primary(KR), International(US/JP/CN) 4개 Market 생성, 각 Market에 통화·언어·도메인 서브패스 매핑 (ople.com/kr, /en, /jp, /cn)
  2. 통화 설정 — Local currencies 활성화, 가격 반올림 룰 정의 (JPY는 소수점 없음, KRW는 100원 단위)
  3. Geolocation 앱 설치 — 방문자 IP 기반 자동 리다이렉트 + 상단 국가/언어 셀렉터 배너

6-2. 다국어 콘텐츠 플로우

  1. Translate & Adapt 앱으로 테마 텍스트·상품명·설명·metafield 자동 번역 초안 생성
  2. 커스텀 앱에 번역 동기화 잡 — 관리자가 커스텀 앱 UI에서 번역본 리뷰/수정 → Admin API로 Shopify에 저장
  3. URL 구조 — hreflang 자동 삽입, Liquid에서 {{ request.locale.iso_code }} 기반 canonical 분기

6-3. 글로벌 배송 & 관세 구현

  1. Shipping Zones 재편 — 국내(CJ대한통운), 동아시아(JP/CN 해외특송), 북미(DHL/UPS) 로 존 분리 및 weight-based rate 설정
  2. Carrier Service API — 커스텀 앱이 외부 운송사 API와 통신하여 실시간 배송료 반환, 체크아웃 화면에 주입
  3. HS코드 & 관세 매핑 — 상품별 HS코드 metafield 등록, 앱이 Shopify Markets Duties API로 사전 관세 계산 → PDP 및 장바구니에 DDP 금액 표시
  4. 결제수단 라우팅 — Shopify Payments(해외 카드) + PortOne(국내 카카오페이/네이버페이/무통장) + PayPal(해외 대체)
